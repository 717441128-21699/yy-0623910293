# 景区应急联动中心 - 后端告警服务

面向成熟景区已有系统的轻量级告警触发器，用于在客服系统、广播系统、领导短信之间形成统一的舆情告警闭环。

## ✨ 三项核心能力

### 1. 风险规则配置
- 工作人员通过简单表单配置 **重点词 + 组合条件**（踩踏、缆车停运、孩子走失、退票聚集等）
- 指定 **责任部门**、**建议核实动作**
- 支持 **OR/AND** 组合逻辑，**次数/时间窗口** 双阈值
- 预置 5 条景区常见规则（踩踏、缆车、走失、退票、设施安全）

### 2. 三级分级告警
| 等级 | 标识 | 触发场景 | 推送目标 |
|------|------|----------|----------|
| 一般关注 | `notice` | 阈值低频次出现 | 值班群 |
| 需核实 | `verify` | 达到中等级阈值 | 责任部门接口 + 值班群 |
| 紧急处置 | `emergency` | 高频次或高危规则 | 领导短信 + 部门 + 值班群 |

告警消息含：原文摘要、来源平台、疑似地点、匹配次数、建议核实动作。
同规则在时间窗口内自动**聚合累加**，不重复推送；持续触发时**自动升级**。

### 3. 告警闭环回填
- 接收部门 **7 种回填状态**：已联系、已到场、现场正常、启动预案、已升级、误报、已办结
- 回填后自动流转告警状态，**已关闭/核实正常 的告警自动避免重复催办**
- 每次回填留痕，完整记录处置轨迹 + 操作人

## 🚀 快速启动

```bash
# 1. 安装依赖
npm install

# 2. 启动服务（首次启动自动建表 + 初始化 5 条预置规则）
npm start

# 服务地址：http://localhost:3000
```

服务启动后自动完成初始化，无需手动建库。

## 📡 模块与接口

### 规则管理 `/api/rules`
| 方法 | 接口 | 说明 |
|------|------|------|
| GET | `/api/rules` | 规则列表（分页 + 状态过滤） |
| POST | `/api/rules` | 新增规则 |
| GET | `/api/rules/:id` | 规则详情 |
| PUT | `/api/rules/:id` | 修改规则 |
| PATCH | `/api/rules/:id/status` | 启停切换（status: 0/1） |
| DELETE | `/api/rules/:id` | 删除规则 |

### 告警记录与分级推送 `/api/alerts`
| 方法 | 接口 | 说明 |
|------|------|------|
| POST | `/api/alerts/ingest` | **消息接入（核心）** 接收单条舆情，匹配规则生成告警 |
| POST | `/api/alerts/batch-ingest` | 批量接入消息 |
| GET | `/api/alerts` | 告警列表（状态/等级/部门过滤 + 分页） |
| GET | `/api/alerts/:id` | 按 ID 查询告警详情（含回填历史） |
| GET | `/api/alerts/uuid/:uuid` | 按 UUID 查询告警详情（**回填接口推荐**） |
| GET | `/api/alerts/statistics/summary` | 统计概览（等级分布/部门分布/今日指标） |

`/alerts/ingest` 响应中的 `push_payloads` 数组可直接用于下游推送（值班群/短信/部门接口）。

### 告警闭环回填 `/api/callbacks`
| 方法 | 接口 | 说明 |
|------|------|------|
| GET | `/api/callbacks/status-options` | **回填状态字典（前端必调）** |
| POST | `/api/callbacks` | **提交回填（核心）** 部门处理后上报状态 |
| GET | `/api/callbacks/alert/:id` | 按告警 ID 查询回填历史 |
| GET | `/api/callbacks/alert-uuid/:uuid` | 按告警 UUID 查询回填历史 |
| GET | `/api/callbacks/recent` | 近期回填记录（默认 50 条） |

## 🔌 典型集成流程

```
客服工单系统 ──┐
游客舆情系统 ──┼──► POST /api/alerts/ingest ──► 告警引擎（匹配+阈值+聚合）──► 值班群/短信/部门接口
广播系统     ──┘                                                      │
                                                                      │ 回填
                                                                      ▼
OA/值班群 ─────────────────────► POST /api/callbacks ───────────► 更新状态 + 避免重复催办
```

## 🗂️ 技术栈

- **运行时**：Node.js (Express 4)
- **数据库**：SQLite（sql.js WASM 实现，零编译依赖，跨平台直接运行）
- **特性**：自动持久化、优雅关闭、参数校验、CORS 支持
